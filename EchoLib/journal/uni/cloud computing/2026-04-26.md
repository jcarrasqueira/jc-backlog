# Cloud Computing — Phase 4 Implementation (Updated)

**Author:** Joana Carrasqueira, 64414

**Branch:** `jcarrasqueira`

## Overview

This phase focused on the implementation of two core services: the **Review Service** and the **Recommendation Service**. A key highlight of this phase is the **inter-service communication**, where the Recommendation Service consumes data from the Review Service via **gRPC** to build accurate user profiles.

## Microservices & Inter-service Communication

### 1. Review Service

Manages the lifecycle of user feedback.

- **Responsibilities:** CRUD operations for ratings and reviews, storage of sentiment/topic metadata.
    
- **Key Feature:** Implements a gRPC server that allows other services to fetch user-specific rating data efficiently.
    

### 2. Recommendation Service

Generates personalized content suggestions using a hybrid scoring algorithm.

- **Data Sources:** Combines internal data (Genre Preferences, Reference Movies) with external data (User Ratings fetched from the Review Service).
    
- **Communication:** Acting as a gRPC client, it calls the Review Service's `get_user_ratings` method to factor user history into the recommendation logic.

## The Recommendation Algorithm Logic

The implemented logic for `get_recommendations` uses a weighted scoring system:

1. **Data Retrieval:** Fetches ratings from the **Review Service** via gRPC.
    
2. **Filtering:** Only positive ratings ($\ge 3.0$) are considered.
    
3. **Scoring Weights:**
    
    - **User Ratings:** The value of the rating is added to the genre's score.
        
    - **Genre Preferences:** Liked genres add **+2.0**; Disliked genres penalize by **-3.0**.
        
    - **Reference Movies:** Genres of reference movies add **+1.5** to the score.
        
4. **Fallback:** If no user data exists, the system executes `get_top_rated_movies` to return the highest-rated global titles.
