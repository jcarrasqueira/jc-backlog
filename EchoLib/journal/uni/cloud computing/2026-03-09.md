## Rating Evaluation Endpoints

| Operation         | Endpoint                               | Description                             |
| ----------------- | -------------------------------------- | --------------------------------------- |
| Get evaluation    | `GET /ratings/{rating_id}/evaluation`  | Retrieve evaluation status for a rating |
| Create evaluation | `POST /ratings/{rating_id}/evaluation` | Mark a rating as valid/invalid          |
| Update evaluation | `PUT /ratings/{rating_id}/evaluation`  | Update evaluation status                |

```yaml
  /ratings/{rating_id}/evaluation:
    get:
      summary: Retrieve rating evaluation
      operationId: getRatingEvaluation
      tags:
        - "Review System"
      parameters:
        - $ref: "#/components/parameters/rating_id"
      responses:
        '200':
          description: Rating evaluation retrieved successfully
          content:
            application/json:
              schema:
                type: object
                properties:
                  rating_id:
                    type: integer
                  valid:
                    type: boolean
                  evaluated_at:
                    type: string
                    format: date-time

    post:
      summary: Create rating evaluation
      operationId: createRatingEvaluation
      tags:
        - "Review System"
      parameters:
        - $ref: "#/components/parameters/rating_id"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - valid
              properties:
                valid:
                  type: boolean
      responses:
        '201':
          description: Rating evaluation created successfully

    put:
      summary: Update rating evaluation
      operationId: updateRatingEvaluation
      tags:
        - "Review System"
      parameters:
        - $ref: "#/components/parameters/rating_id"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - valid
              properties:
                valid:
                  type: boolean
      responses:
        '200':
          description: Rating evaluation updated successfully
```


## User Preferences Endpoints

| Operation            | Endpoint                                      | Description                  |
| -------------------- | --------------------------------------------- | ---------------------------- |
| Get user preferences | `GET /users/{user_id}/preferences`            | List preferred genres/topics |
| Add a preference     | `POST /users/{user_id}/preferences`           | Add a new preferred genre    |
| Remove a preference  | `DELETE /users/{user_id}/preferences/{genre}` | Remove a preferred genre     |

```yaml
  /users/{user_id}/preferences:
    get:
      summary: Retrieve user preferences
      operationId: getUserPreferences
      tags:
        - "User Preferences"
      parameters:
        - $ref: "#/components/parameters/user_id"
      responses:
        '200':
          description: User preferences retrieved successfully
          content:
            application/json:
              schema:
                type: array
                items:
                  type: string

    post:
      summary: Add a user preference
      operationId: addUserPreference
      tags:
        - "User Preferences"
      parameters:
        - $ref: "#/components/parameters/user_id"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required:
                - genre
              properties:
                genre:
                  type: string
      responses:
        '201':
          description: Preference added successfully

  /users/{user_id}/preferences/{genre}:
    delete:
      summary: Remove a user preference
      operationId: deleteUserPreference
      tags:
        - "User Preferences"
      parameters:
        - $ref: "#/components/parameters/user_id"
        - name: genre
          in: path
          required: true
          schema:
            type: string
      responses:
        '204':
          description: Preference removed successfully

```
