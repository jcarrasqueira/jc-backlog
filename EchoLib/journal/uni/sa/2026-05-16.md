#### 1. extract header hashes of logs
```python
import sys
import os
 
def extract(path):
    with open(path, 'rb') as f:
        raw = f.read()
    salt     = raw[:16]           # bytes 0-15
    verifier = raw[16:48]         # bytes 16-47
    msg      = salt + b'verify'   # what G16 HMACs over
    return verifier.hex() + ':' + msg.hex()
 
files = sys.argv[1:] if len(sys.argv) > 1 else ['gallerylog1', 'gallerylog2', 'gallerylog3']
 
print("Extracting needed hashes from header...")
print()

for path in files:
    if not os.path.exists(path):
        print(f"# SKIP: {path} not found")
        continue
    target = extract(path)
    print(f"# {path}")
    print(target)
    print()
```

#### 3. Apply hashcat

##### explaination
```bash
hashcat -m 1450 -a 3 g16_hashes.txt ?1?1?1?1?1?1?1 -1 ?l?u?d --hex-salt
```

- **`-m 1450`** — the hash mode. Tells hashcat what algorithm to use. 1450 = HMAC-SHA256, which matches exactly what G16 uses in `crypto.py`.\
- **`-a 3`** — the attack mode. Mode 3 = brute-force mask attack, meaning it generates every possible combination itself rather than reading from a wordlist.
- **`g16_hashes.txt`** — your input file with the extracted hashes in `hash:salt` format, one per line.
- **`?1?1?1?1?1?1?1`** — the mask, i.e. the pattern of the password to guess. Each `?1` is one character position. Seven `?1`s means a 7-character token. Hashcat will try every combination of your custom charset in each position.
- **`-1 ?l?u?d`** — defines what charset `?1` refers to. This is a custom charset built from three of hashcat's built-ins:
- `?l` = lowercase letters (a-z)
- `?u` = uppercase letters (A-Z)
- `?d` = digits (0-9)
Combined, `-1 ?l?u?d` means "charset 1 = all alphanumeric characters", which matches the spec since the project says tokens are alphanumeric.
**`--hex-salt`** — tells hashcat the salt field in your hash file is hex-encoded, not plain text. You need this because the message contains the raw 16 binary bytes of the salt, which can't be represented as ASCII.

**To run hashcat on your machine**, tell it explicitly to use both CPU and iGPU:

```bash
hashcat -m 1450 -a 3 g16_hashes.txt ?1?1?1?1?1?1?1 -1 ?l?u?d --hex-salt -D 1,2
```

The `-D 1,2` flag enables both CPU (device 1) and OpenCL GPU (device 2) — on integrated graphics this squeezes out a bit more performance than either alone.
