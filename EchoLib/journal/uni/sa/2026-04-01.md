# Ideas for security
## Protection for IV reuse
###  1. Add iv derivation 
- don't use urandom can repeat iv
- security intuition:
    - k_mac is secret (derived from token + salt).
    - entry_index is public but strictly increasing.
    - HMAC gives you pseudorandom, non‑repeating IVs under the same key.

```python
import struct, hmac, hashlib

def derive_iv(k_mac: bytes, entry_index: int) -> bytes:
    """
    Deterministically derive a unique IV for each log entry.
    IV_n = HMAC(k_mac, b"iv" || uint32_be(entry_index))[:16]
    """
    counter_bytes = struct.pack("!I", entry_index)  # 4-byte big-endian counter
    return hmac.new(k_mac, b"iv" + counter_bytes, hashlib.sha256).digest()[:16]
```

```python
def append_entry(path: str, token: str, plaintext: bytes):
    with open(path, 'r+b') as f:
        # 1) Read header
        salt = f.read(16)
        mac_prev = f.read(16)

        # 2) Derive keys
        k_encryption = keygen(token, salt, b'enc')
        k_mac        = keygen(token, salt, b'mac')

        # 3) Scan existing entries to:
        #    - verify MAC chain
        #    - count how many entries exist
        entry_index = 0
        while True:
            iv = f.read(16)
            if not iv:
                break

            cipher_len_bytes = f.read(4)
            if len(cipher_len_bytes) < 4:
                raise ValueError("corrupted log")

            cipher_len = struct.unpack("!I", cipher_len_bytes)[0]
            ciphertext = f.read(cipher_len)
            if len(ciphertext) < cipher_len:
                raise ValueError("corrupted log")

            stored_mac = f.read(16)
            if len(stored_mac) < 16:
                raise ValueError("corrupted log")

            correct_mac = hmac.new(
                k_mac,
                mac_prev + iv + ciphertext + cipher_len_bytes,
                hashlib.sha256
            ).digest()[:16]

            if not hmac.compare_digest(correct_mac, stored_mac):
                raise ValueError("corrupted log")

            mac_prev = stored_mac
            entry_index += 1

        # 4) Now entry_index is the index for the *next* entry
        iv = derive_iv(k_mac, entry_index)

        # 5) Encrypt plaintext
        ciphertext = encrypt(k_encryption, iv, plaintext)
        cipher_len_bytes = struct.pack("!I", len(ciphertext))

        # 6) Compute MAC for new entry
        mac_new = hmac.new(
            k_mac,
            mac_prev + iv + ciphertext + cipher_len_bytes,
            hashlib.sha256
        ).digest()[:16]

        # 7) Append new entry
        f.write(iv)
        f.write(cipher_len_bytes)
        f.write(ciphertext)
        f.write(mac_new)
```

### 3. Add iv-reuse detection
```python
def verify_chain(path: str, token: str):
    with open(path, 'rb') as f:
        salt = f.read(16)
        mac_prev = f.read(16)

        k_encryption = keygen(token, salt, b'enc')
        k_mac        = keygen(token, salt, b'mac')

        entry_index = 0

        while True:
            iv = f.read(16)
            if not iv:
                break

            # 1) Check IV matches derived IV for this index
            expected_iv = derive_iv(k_mac, entry_index)
            if not hmac.compare_digest(iv, expected_iv):
                raise ValueError("corrupted log")

            cipher_len_bytes = f.read(4)
            if len(cipher_len_bytes) < 4:
                raise ValueError("corrupted log")

            cipher_len = struct.unpack("!I", cipher_len_bytes)[0]
            ciphertext = f.read(cipher_len)
            if len(ciphertext) < cipher_len:
                raise ValueError("corrupted log")

            stored_mac = f.read(16)
            if len(stored_mac) < 16:
                raise ValueError("corrupted log")

            correct_mac = hmac.new(
                k_mac,
                mac_prev + iv + ciphertext + cipher_len_bytes,
                hashlib.sha256
            ).digest()[:16]

            if not hmac.compare_digest(correct_mac, stored_mac):
                raise ValueError("corrupted log")

            mac_prev = stored_mac
            entry_index += 1

        return k_encryption, k_mac, mac_prev

```

### project justification
#### threat
AES‑CTR IV reuse allows attackers to recover plaintext XOR differences, breaking confidentiality.
#### mitigation
We derive IVs deterministically as `IV_n = HMAC(k_mac, "iv" || n)[:16]` ensuring uniqueness and preventing attacker‑controlled IVs.

## Libs
### Struct
- Used for: packing and unpacking binary integers
#### why
- Your log format uses binary fields, not text.
- struct lets you convert between Python integers and fixed‑size byte sequences.

```
struct.pack("!I", n)     # 4‑byte unsigned integer, big‑endian
struct.unpack("!H", b)   # 2‑byte unsigned integer, big‑endian
```

| Notation | Meaning            | Size    | Used For                                          |
| -------- | ------------------ | ------- | ------------------------------------------------- |
| ``!``    | Network byte order | —       | Ensures consistent binary format                  |
| ``I``    | Unsigned int       | 4 bytes | Ciphertext length, counters, timestamps, room IDs |
| ``H``    | Unsigned short     | 2 bytes | TLV field lengths                                 |
| ``!I``   | Big‑endian uint32  | 4 bytes | ``[ ``4 ``bytes ``] ``LEN``                       |
| ``!H``   | Big‑endian uint16  | 2 bytes | ``[ ``2 ``bytes ``] ``LENGTH`` in TLV             |
### cryptography.hazmat.primitives.ciphers
- Used for: AES‑CTR encryption and decryption
```python
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
```

#### Why it’s needed
- `Cipher` constructs an encryption/decryption object.
- `algorithms.AES(key)` selects AES as the block cipher.
- `modes.CTR(iv)` selects CTR mode, which turns AES into a stream cipher.

#### Why CTR mode?
- No padding required
- Random access to ciphertext
- Perfect for logs where each entry is independent
- Fast and widely supported
This directly supports the project requirement that the system must _“guarantee the confidentiality… of the log”_.

### hmac
- Used for: Message Authentication Codes (MACs)
``` python
import hmac
```

#### why
- HMAC ensures integrity and tamper detection.
- You use it to build a MAC chain, which prevents:
	- log modification
	- entry deletion
	- entry reordering
	- entry insertion
- This aligns with the project requirement that an attacker must not be able to “modify the log… or fool logread into accepting a bogus file.”

### hashlib
- Used for: SHA‑256 inside HMAC
#### why
SHA‑256 is:
- secure
- widely used
- collision‑resistant
- appropriate for MAC chaining
You truncate the output to 16 bytes (128 bits), which is still secure for this context. -> so AES-128 can be used instead AES-256
